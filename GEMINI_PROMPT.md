# MemeticChaos 项目现状（给外部 AI 的求助）

## 一句话

中国互联网集体情感的混沌属性建模。两层架构：外部场（平台注意力数据）+ 内部叙事层（叙事结构）。
想预测的不是"下一个爆款梗"，是集体情绪系统的秩序形态演化。

## 数据资产（已有，不必担心数据来源）

```
外部场:      51 关键词 × 132 月 = 5217 数据点 (Google Trends 2015-2025)
内部层:      43 梗 × 132 月 = ~1048 数据点
叙事:        87 条 (22 B站视频转录 + 36 Google Trends曲线生成 + 29 人工策展)
LLM 概念分数: 64 条叙事 × 35 维 → 5D 约束场
集体混沌轴:  133 个月 (2015-01 → 2026-07)
实时采集:    微博50 + 百度50 + 知乎30 = 130条/小时 (服务器 24/7)
Dashboard:   Flask + ECharts, chaos.mote-pal.xyz
```

## 当前模型的运作方式（v0.2，已确认有问题）

每月构建一个状态向量，包含：

1. **混沌轴**：把当月活跃梗按五个类别（解构自嘲/攻击发泄/虚无退却/身份认同/纯粹娱乐）
   的固定权重（-0.33, -0.62, -0.59, +0.34, +0.19）做注意力加权平均。
   五个权重是人肉标的，无数据支撑。

2. **约束场 5D**：LLM 对每个梗的叙事打 35 个概念分（如"是否包含愤怒""是否是草根自发"等），
   聚合成 Identity/Humor/Conflict/Novelty/Accessibility 五个维度。
   **关键问题：每个梗只打了一次分，不随时间变化。** 同一个梗在 2021 年爆发期
   和 2024 年消退期用同一个约束向量。

3. **类别分布**：五类别的注意力占比。类别是人工标注的，互斥（一个梗只能一个类）。
   但实际中一个梗可以同时属于多个类别。

4. **注意力结构**：HHI（集中度）+ 类别熵（多样性）。

用过去 6 个月的 [外部场_PCA + chaos + constraint + cat_dist + hhi + entropy]
→ Ridge → 预测下个月的 [chaos, constraint, cat_dist, hhi, entropy]。

**内部叙事层**用过去 6 个月的 constraint 向量 → Ridge → 预测 Layer 1 的混沌残差。

## 已确认的问题（2026-07-04 审计）

**结构性缺陷**：

1. **约束场是静态标签**。每个梗只有一个 5D 向量。月度聚合的变化仅来自
   "哪些梗活跃"的名单变化，变化幅度极小。叙事层从未真正参与预测。

2. **内部层被偷换成约束向量**。本应使用 87 条叙事的结构信息（传播阶段数、
   变异类型、语义漂移方向），实际压缩成了 5 个数字。内部层贡献为 0。

3. **混沌轴权重是人肉编的**。五个类别映射值没有数据支撑。混沌轴 ≈
   类别固定权重 × 活跃梗注意力加权平均——本质上是个人工定义的指标。

4. **类别互斥且人工标注**。一个梗只能分到一个类别（如"普信男"="攻击发泄"），
   但它同时属于"解构自嘲"。多标签信息被丢弃了。HHI/熵的计算依赖这个分类。

5. **月度粒度可能太粗**。混沌轴月间预测 MAE 比 lag-1 差 22%——是随机游走。
   但不确定是因为它真的随机，还是因为周度/日度有结构、月间采样频率太低。

**数据质量问题**：

6. LLM 概念打分平均 16/35 概念为零分，24% 的梗超过 20 个零。
   约束场是分组均值，零分把约束压向零，人为降低方差。

7. Dashboard 当前月（2026-07）混沌轴 +0.229 仅基于 2 个梗 8 条 scraper 信号。

**实证结论（非 bug）**：
- 混沌轴是随机游走——符合"与混沌共存"的预期
- 注意力结构 R²=0.79（HHI+熵可从外部场预测）
- 叙事概念向量聚类 ARI=0.27（机器见 3 类，非人工 5 类）

## 想要的方向

### 外部层（可能不需要大改）

外部场 51 关键词 PCA→8 维，已经能预测注意力结构（R²=0.79）。
这部分可能不需要重构，但需要换掉人肉编的混沌轴。

### 内部叙事层（需要从零重想）

当前"约束场"是错的——它是梗的静态属性，不是叙事层。

87 条叙事有丰富的结构信息：
- 传播阶段（origin → emergence → peak → controversy → fixation）
- 变异类型（戏仿、反讽、语义漂移、二次创作、制度化收编）
- 语义漂移方向和幅度
- 关键触发事件的类型（政策、明星、偶然、草根自发、平台活动）

这些结构信息**随时间变化**——同一个梗在不同阶段有不同的叙事特征。
而且这些信息可以直接从 LLM 抽取的叙事 JSON 中提取，不需要额外的概念打分步骤。

**想法**：用叙事结构特征（而非 5D 约束向量）作为内部层的输入。
叙事结构的月度聚合可能比静态约束场的月度聚合更有预测价值。

### 约束场应该动态化

如果保留约束场概念，它不应该是"一个梗一个向量"。应该是：
- 每个梗在不同阶段有不同的约束值
- 或者：约束场的月度变化来自哪些叙事类型在主导，而不是哪些梗在活跃

### 五类别应该允许多标签或被数据驱动替代

叙事概念向量的无监督聚类已经找到了 3 种类型（ARI=0.27 vs 人工 5 类）。
机器发现的 3 种类型可能比人工 5 类更有信息量。或者允许多标签分类。

### 更高的时间分辨率

scraper 每小时有 130 条数据。如果能聚合到周度/日度，
混沌轴在更高分辨率下可能不再显示为随机游走。

## 核心困惑（请你帮忙想的）

1. **外部层和内部层的耦合方式是什么？** 当前架构是"外部场→基线，内部叙事→修正残差"。
   但如果叙事层提供的是完全不同的信息（叙事结构 vs 平台注意力），
   也许不应该用"残差修正"这种加法形式。更好的耦合方式是什么？

2. **叙事结构怎么变成时间序列？** 87 条叙事的结构信息本质上是"梗的传播学特征"。
   但每个月需要的是一个集体叙事状态向量。怎么从 87 条个体叙事聚合出月度集体叙事状态？
   是通过当前活跃梗的叙事特征加权？还是另建一套独立的月度叙事指标？

3. **混沌轴到底是做什么的？** 它目前是人肉编的指标，但作为系统的核心输出变量，
   它的定义应该更严格——要么用数据驱动的方式重新定义（比如叙事概念向量的第一主成分），
   要么放弃它，直接用秩序形态分类作为输出。

4. **预测什么最有意义？** 如果混沌轴值不可预测（随机游走），那么应该预测什么？
   秩序形态的分类？转移概率？约束场的变化方向？叙事类型的分布迁移？
