# MemeticChaos 项目总结（给外部 AI）

## 项目目标

从外部数据（平台注意力场 + 叙事结构）学习中国互联网集体情感系统的演化规律，
预测未来 1-6 个月的主导秩序形态。

不是预测下一个爆款梗，是预测集体情绪系统会以什么形态组织自己。
不是论文，是功能系统。AlphaGo 范式：不灌输人类套路。

## 当前数据 (2026-06-30)

```
外部环境场: 51 关键词 × 132 月 = 5217 数据点
  来源: Google Trends 2015-2025 (CN)

内部梗层: 43 梗 × 133 月 = 133 月混沌轴 + 类别分布
  每个梗有: 月度注意力曲线 + LLM 35维概念分数 + 5维约束场

叙事层: 87 条叙事 (22 B站 + 36 曲线生成 + 29 策展)
  其中 57 条已完成 LLM 35维概念打分

实时采集: 微博50 + 百度50 + 知乎30 = 130条/小时, 服务器 24/7

集体混沌轴: 133 个月 (2015-01 → 2026-06)
  10 次系统级相变检测
  最新月 (2026-06) 由 scraper 数据估算
```

## 已完成的模型 (FR19 v0.2)

**两层预测架构**：
```
外部场 (51维 PCA→8维) + 约束场(5维) + 类别分布(5维) + HHI + 叙事熵
  → RidgeCV → 混沌轴 + 约束场 + 类别分布 + 注意力结构预测
  → KMeans 聚类 → 8 种秩序形态
```

**关键发现**：
1. 混沌轴是随机游走（月间预测 MAE 比 lag-1 差 25%）— 确认"与混沌共存"
2. 秩序形态有结构：注意力结构 R²=0.79，约束场 R²=0.52
3. LLM 概念打分替代软匹配后约束场方差提升 15-22x
4. 叙事无监督聚类：机器发现 3 类，非人工 5 类（ARI=0.27）
5. **内部叙事层贡献仍是 0**：约束场方差有了，架构有了，还没转化成预测能力

**Dashboard**: Flask + ECharts, 9 个 API 端点, token auth, 历史回放, 精细建模查询
**数据管线**: 每小时采集 → 每天新梗发现+叙事生成+概念打分 → 每周预测报告+月度叙事摘要

## 当前卡点

1. **内部叙事层无法激活**：约束场信息已经存在（R²=0.52），但加到预测器里贡献为 0。
   可能原因：(a) 约束场是梗的静态属性，月度聚合后变化太慢
   (b) 月度时间分辨率太低，约束场的变动在月间尺度上看不到
   (c) 约束场对混沌轴的调制是非线性的，Ridge 抓不住

2. **混沌轴随机游走**：是数据问题（132 月太少）还是结构问题（混沌轴本身就是 I(1) 过程）？
   如果提高到周度/日度分辨率，是否能看到均值回归或短期可预测性？

3. **预测什么才有意义**：如果混沌轴值不可预测，那么"秩序形态"（分类）的预测准确率是多少？
   形态之间的转移矩阵是否稳定？

## 下一步方向

1. **更高时间分辨率**：scraper 已有小时级数据 → 聚合到周度/日度，
   看混沌轴在更细粒度上是否有可预测结构

2. **叙事类型作为预测特征**：机器发现的 3 种叙事类型（ARI=0.27 vs 人工 5 类）
   加入预测器，可能比 5 维约束场更有信息量

3. **非线性模型**：约束场 → 混沌轴的调制可能不是 Ridge 能捕获的。
   试 GradientBoosting / 简单 MLP？

4. **FR31 — 情感约束场顾问**：将 MemeticChaos 的集体规律作为参考层，
   构建个体层面的情感咨询系统。三层架构：
   - Layer 1: 集体涌现模式（参考基线）
   - Layer 2: 个体画像 + 互动历史
   - Layer 3: 当前情境 + 情感温度
